# Visual References

This file is a lightweight holding area for visual direction research for Gradient.

The goal is not to become design experts before moving forward. The goal is to collect a small set of strong references that we can later hand to a visual UX agent and say: "Use these as inspiration for a few polished theme directions for Gradient."

## Current Visual Snapshot

From the current app styling in `src/styles.css`, Gradient already leans toward:

- warm dark background
- glassy panels
- gold / sand accent tones
- rounded surfaces
- soft premium-game energy

What still feels under-authored:

- typography is generic
- spacing and scale feel less systemized than they could
- glass effects are doing more of the branding work than the type, shape, and layout

## Lowest-Effort Workflow

If we want the simplest possible process, use this:

1. Browse examples on a reference site.
2. Save 3 to 5 examples that feel close to the direction we want.
3. Write a short note for each reference using the template below.
4. Later, ask a visual UX agent to create 2 to 3 theme concepts for Gradient based on those references.
5. The agent should extract principles, not copy layouts directly.

This means we do not need to deeply study fonts, spacing systems, or color theory up front.

Link: <https://mobbin.com/>

Why it is useful:

- best source for real product UI and mobile app screens

- useful when we want believable app structure, controls, menus, cards, onboarding, settings, and dashboards

## Best Supporting Tools

These are not the first thing to browse if we just want references, but they become useful once we start turning reference picks into a theme system.

### Typography

- Typewolf: <https://www.typewolf.com/>
  Good for understanding contemporary web typography and font pairings.
- Fonts In Use: <https://fontsinuse.com/>
  Good for seeing how fonts behave in real projects.
- Fontshare: <https://www.fontshare.com/>
  Good source of free fonts with more personality than typical defaults.
- Google Fonts: <https://fonts.google.com/>
  Good for production-safe testing, especially when we want easy licensing.

### Color

- Radix Colors: <https://www.radix-ui.com/colors>
  Accessible UI color scales, especially useful for dark themes.
- Adobe Color: <https://www.adobe.com/products/color.html>
  Useful for exploring palettes and extracting themes from imagery.
- Coolors: <https://coolors.co/>
  Fast for rough palette generation and exploration.

### Scale, Spacing, and System Thinking

- Utopia: <https://utopia.fyi/>
  Helpful for fluid type and spacing scales.
- IBM Design Language / 2x Grid: <https://www.ibm.com/design/language/>
  Good reference for disciplined spacing and layout rhythm.
- Carbon Design System: <https://carbondesignsystem.com/>
  Good for structured, professional product UI thinking.
- Apple Human Interface Guidelines: <https://developer.apple.com/design/human-interface-guidelines/>
  Good for hierarchy and refined interaction decisions.
- Material 3: <https://developer.android.com/develop/ui/compose/designsystems/material3>
  Good for type scale, shape tokens, and interface system patterns.

## What To Notice In A Reference

No deep theory needed. When looking at a screen, just ask:

- Type: does it feel technical, calm, playful, editorial, premium, soft, sharp?
- Color: muted or vivid, warm or cool, soft contrast or high contrast?
- Scale: compact or spacious, big headlines or quieter hierarchy?
- Shape: sharp corners, rounded cards, pills, circles, square frames?
- Surface: flat, layered, glassy, tactile, metallic, paper-like?
- Density: tool-like, dashboard-like, gallery-like, game-like?

If we can answer those six questions, we already have enough signal to guide a visual UX agent later.

## What Probably Fits Gradient

Gradient is a puzzle app, so strong references will likely come from a mix of:

- polished productivity apps
- premium lifestyle apps
- games with restrained UI
- editorial product sites with strong color and type

Good search directions when browsing:

- game UI
- collection or library screens
- settings panels
- stats or progress screens
- cards
- premium dark UI
- bold typography
- tactile controls

## Recommended First Pass For Gradient

Try to save one reference in each of these buckets:

1. Clean product polish
   Something that feels clear, premium, and credible.
2. Tactile playful interface
   Something that makes controls and tiles feel satisfying.
3. Bold visual identity
   Something with stronger typography, color, or art direction.

That gives us a useful triangle of references instead of locking us into one exact clone.

## Prompt For A Future Visual UX Agent

Use the references in `visual_references.md` as inspiration for Gradient. Create 2 to 3 distinct visual theme directions for this puzzle web app. Do not copy layouts directly. Instead, extract principles from the references for typography, palette, spacing, card styling, shape language, and overall interface mood. Keep the result implementable in the existing app.

## Practical Recommendation

If we want the fastest path with the least design homework:

1. If paid is fine, use Mobbin first.
2. If paid is not fine, use Banani first.
3. Use Refero or Land-book second.
4. Use Godly only to add more personality.
5. Save 3 to 5 references.
6. Then ask the visual UX agent to turn them into theme directions for Gradient.

That is enough to move forward without getting stuck in design research.
